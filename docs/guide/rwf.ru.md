# Рабочий процесс выпусков (RWF)

Модуль **Рабочий процесс выпусков** добавляет в EA-ландшафт **управление изменениями на основе веток**. Все изменения предлагаются в именованной *ветке*, проверяются архитектором и применяются к активному ландшафту только после утверждения.

!!! note
    Модуль Рабочий процесс выпусков может быть включён или отключён администратором в разделе [Настройки](../admin/settings.md). При отключении элемент навигации «Releases» и все конечные точки веток скрываются.

## Обзор

| Понятие | Описание |
|---------|----------|
| **Ветка** | Именованная изолированная среда. Карточки, связи и диаграммы можно редактировать в полной изоляции от активного ландшафта |
| **Рабочее пространство** | Представление, ограниченное веткой (вкладки: Карточки, Связи, Диаграммы) |
| **Diff** | Сравнение полей между черновиком ветки и текущим активным ландшафтом с обнаружением конфликтов |
| **Слияние** | Применение изменений ветки к активному ландшафту после утверждения |
| **Снимок** | Неизменяемая копия полного ландшафта на определённый момент времени, используемая как базовая линия |

### Гарантия изоляции

**Ветки не влияют ни на одно существующее представление.** Отчёты, BPM, PPM, GRC, TurboLens, диаграммы, инвентарь и детали карточек всегда отображают активный ландшафт.

## Разрешения

| Разрешение | Для кого | Что позволяет |
|-----------|---------|---------------|
| `rwf.view` | Все пользователи, которым нужно видеть ветки | Читать список веток, представление diff и список снимков |
| `rwf.contribute` | Участники | Создавать ветки, редактировать карточки/связи/диаграммы в ветке, отправлять на проверку |
| `rwf.approve` | Архитекторы EA / рецензенты | Утверждать, отклонять и выполнять слияние веток в активный ландшафт |

## Жизненный цикл ветки

```
открыта  →  на проверке  →  утверждена  →  слита
                        ↘  отклонена    ↗
                        ↘  заброшена
```

## Создание ветки

1. Перейдите в **Releases → Ветки**.
2. Нажмите **Новая ветка**.
3. Введите название и необязательное описание, затем нажмите **Создать ветку**.

## Работа в рабочем пространстве

Откройте ветку и нажмите **Открыть рабочее пространство**. Рабочее пространство имеет три вкладки:

### Вкладка Карточки

Показывает все карточки, видимые в этой ветке, с их статусом изменения (Новая / Изменена / Удалена). Нажмите на строку карточки, чтобы открыть панель деталей в рамках ветки.

### Вкладка Связи

Показывает все связи, видимые в этой ветке, с их статусом изменения.

### Вкладка Диаграммы

Перечисляет все переопределения диаграмм VisualFirst в этой ветке. Нажатие на строку открывает полный редактор VisualFirst в режиме ветки.

## Отправка на проверку

Откройте страницу деталей ветки и нажмите **Отправить на проверку**. Все пользователи с `rwf.approve` получат уведомление.

## Проверка и утверждение

1. Перейдите в **Releases → Ветки** и выберите ветку на проверке.
2. Нажмите **Просмотр изменений**, чтобы увидеть полный diff.
3. Нажмите **Утвердить** или **Отклонить** (с необязательным комментарием).

## Слияние с активным ландшафтом

После утверждения нажмите **Слить в основной**. При наличии конфликтов появится диалог разрешения конфликтов.

## Откат

Слитые ветки поддерживают действие **Отменить слияние**, которое восстанавливает состояние до слияния.

!!! warning
    Откат нельзя отменить.

## Снимки

Снимки — это именованные неизменяемые копии полного ландшафта на определённый момент времени.

1. Перейдите в **Releases → Ветки → вкладка Снимки**.
2. Нажмите **Новый снимок**, введите название и нажмите **Создать снимок**.
3. Нажмите **Просмотр изменений**, чтобы увидеть, что изменилось в ландшафте с момента создания снимка.
