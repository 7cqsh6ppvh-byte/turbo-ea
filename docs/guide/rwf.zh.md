# 发布工作流（RWF）

**发布工作流**模块为 EA 全景添加了**基于分支的变更治理**。所有变更在具名*分支*中提出，经架构师审核后，仅在批准后才合并到实时全景中。

!!! note
    发布工作流模块可由管理员在[设置](../admin/settings.md)中启用或禁用。禁用后，「Releases」导航项和所有分支端点将被隐藏。

## 概览

| 概念 | 说明 |
|------|------|
| **分支** | 一个具名沙箱。卡片、关系和图表可以完全独立于实时全景进行编辑 |
| **工作区** | 分支范围的视图（选项卡：卡片、关系、图表） |
| **差异** | 分支草稿与当前实时全景之间的逐字段比较，包含冲突检测 |
| **合并** | 批准后将分支变更应用到实时全景 |
| **快照** | 某一时刻完整全景的不可变副本，用作比较基准 |

### 隔离保证

**分支不会影响任何现有视图。** 报告、BPM、PPM、GRC、TurboLens、图表、清单和卡片详情始终显示实时全景。

## 权限

| 权限 | 适用对象 | 功能 |
|------|---------|------|
| `rwf.view` | 所有需要查看分支的用户 | 读取分支列表、差异视图和快照列表 |
| `rwf.contribute` | 贡献者 | 创建分支、在分支中编辑卡片/关系/图表、提交审核 |
| `rwf.approve` | EA 架构师 / 审核者 | 批准、拒绝并将分支合并到实时全景 |

## 分支生命周期

```
开放  →  审核中  →  已批准  →  已合并
              ↘  已拒绝   ↗
              ↘  已放弃
```

## 创建分支

1. 导航至**Releases → 分支**。
2. 点击**新建分支**。
3. 输入名称和可选描述，然后点击**创建分支**。

## 在工作区中工作

打开一个分支，点击**打开工作区**。工作区有三个选项卡：

### 卡片选项卡

显示此分支中可见的所有卡片，以及变更状态（新建 / 已修改 / 已删除）。点击卡片行可打开分支范围的卡片详情面板。

### 关系选项卡

显示此分支中可见的所有关系及其变更状态。

### 图表选项卡

列出此分支中所有 VisualFirst 图表覆盖项。点击某行即可在分支模式下打开完整的 VisualFirst 编辑器。

## 提交审核

打开分支详情页，点击**提交审核**。所有拥有 `rwf.approve` 权限的用户将收到通知。

## 审核与批准

1. 导航至**Releases → 分支**，选择正在审核的分支。
2. 点击**查看变更**以查看完整差异。
3. 点击**批准**或**拒绝**（可附带评论）。

## 合并到实时全景

批准后，点击**合并到主分支**。如果存在冲突，将出现冲突解决对话框。

## 回滚

已合并的分支支持**撤销合并**操作，可将所有卡片、关系和图表恢复到合并前的状态。

!!! warning
    回滚操作无法撤销。

## 快照

快照是某一时刻完整全景的具名不可变副本。

1. 导航至**Releases → 分支 → 快照**选项卡。
2. 点击**新建快照**，输入名称，然后点击**创建快照**。
3. 点击**查看变更**以查看自该快照以来全景发生了哪些变化。
